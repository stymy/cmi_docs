Rocky Data Organization
All data is housed in 
  • /home/data 
Within this folder there are several relevant directories:
 
Incoming
This holds data that is coming in from some other site or location, maybe as an archive. Data in this directory is intended to be here temporarily with the goal of moving it in a more organized fashion into the Originals directory (see below).
 
Originals
For an Example study you would have:
  • /home/data/Originals/Example 
This is where (big surprise) the original data before any processing is kept.
 
PreProc
The PreProc folder holds all the data preprocessed by C-PAC (or other packages). The folder names in this directory should correspond to the ones in the Originals. For example:
  • /home/data/PreProc/Example
This folder will be protected so that only people who is responsible for data preprocessing can write to it. All others can read from it for their analysis.
 
Projects
  • /home/data/Projects/Example 
The Projects folder is where users configure and run their analyses (e.g., group analyses with FSL). Typically, the analyses run in this folder will make use of the preprocessed data in a corresponding folder within PreProc. Scripts, settings, configuration files, temporary files and log files can be saved in the projects folder. For instance, scripts to organize ones data in Originals annd scripts or configuration files to run the preprocessing in PreProc should be kept within the Projects folder. Users are reponsible for the hygiene of their projects folder, and intermediate files should be deleted after analyses are done. 
 
PublicProgram
In this folder, there are programs that could be shared by Rocky the server and the cluster, so that we have the same versions of these software on both computation environment. Pros are: 
  • easy version control
  • easy management (don't need to be sudoer to update them). 
Cons are: 
  • the drive is mounted to the clusters as a NSF
  • might require specific settings of .bashrc file. See template .bashrc
 
Freesurfer?
The Freesurfer folder holds the individual subject output within your individual study directories when running freesurfer's recon-all.
